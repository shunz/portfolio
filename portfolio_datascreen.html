背景


品胜、千机网、数据大屏是什么，之间的关系？

- 品胜是一家3C智能移动周边产品龙头企业，销售手机充电线、电池、充电宝等数码周边配件。

- 千机网是一个3C 数码 O2O 电商平台，是品胜的战略合作伙伴、官方授权服务商，为手机消费者提供主流品牌机型的专业手机维修服务。

- 数据大屏将品胜、千机网后端多个系统中与手机服务相关的业务数据进行统一收集、处理和展示。


问题

- 品胜、千机网的电商业务数据分布在多个后端系统中，难以快速知晓业务数据情况和业务变化趋势


解决方案

- 首先收集各方对业务数据的查询统计需求

- 再根据数据需求设计一个数据平台对业务数据进行统一采集、处理和展示

数据大屏系统界面

我的职责

- 与业务干系人反复沟通，收集、分解需求，讨论确认大屏的功能模块。

- 与各方共同商定项目计划

- 完成产品设计环节，包括开展研究目标用户/用户目标、产品定位思考、设计可交互原型、用户测试验证等，以确定最终用于呈现业务数据信息的相对最优设计方案。

- 跟踪和推动 UI 设计、前后端开发、测试验收、发布上线等各项后续环节，然后进入下一轮问题收集、产品设计迭代。


合作对象

- 业务干系人：与其讨论确认需求和功能模块，定期安排测试和验证，降低各环节跑偏风险。

- UED小组：期间带领 UED 组内的 1 位 UI 设计师、2 位前端完成前期的产品体验设计工作。

- 开发团队：与一个专项开发小组紧密协作，以确保最终上线产品的一致性和质量。

设计过程


受众分析（面向群体）

- 品胜管理层、电商业务部门、运营人员、外部合作者/投资者


后端系统调研

- 经分析，发现业务数据主要在两个大部门的后端系统中：信息技术中心的品胜业务数据、软件中心的千机网业务数据


竞品分析

- 搜索、收集了大量和数据大屏、数据可视化相关的竞品，从中学习了不少和数据大屏有关的数据可视化展现设计模式

遇到的挑战


跨部门协调

- 由于后端系统分布在多个部门，需要沟通、协调、推动多个部门配合，一起打通品胜各个后端信息孤岛


可视化方案技术选型

- 最初业务需求方有一些天马行空的想法，想把业务数据展现成很有科幻感，但考虑到 Deadline、实现成本、开发团队技术能力等方面的约束，最终和需求方、开发团队共同商定采用开源的 ECharts 可视化库


多终端的适配

- 大屏版上线后，老板们用的很开心，然后就开心的再提了更多的需求，其中包括希望在外也能通过笔记本、手机、平板演示数据大屏，于是就有了移动端版本。受限于移动设备的特点（比如尺寸更小，不适合展示过多数据维度），对原有一些模块做了精简或调整

小结


产品影响力

- 数据大屏不但方便公司内部了解业务数据情况和业务变化趋势

- 还方便了品胜对外招商展示，促进与外部合作项目的营销推广

- 同时也暴露品胜后台业务系统的一些问题，倒推业务系统优化完善


有哪些收获

- 虽然以前也做过一些数据可视化的项目，但通过这次电商业务数据大屏产品的设计和实现，再次学习了一些大屏设计过程中的方法和原则，为后续数据可视化类产品的设计又提供了一些借鉴思路。

- 数据大屏属于数据可视化范畴，应用场景很广，是目前常用来分享、沟通、传播信息的有效途径之一。它已进化成一种新的媒体形式，在品牌推广、政务接待、商业沟通、数据监控等各个场景发挥重要作用。
